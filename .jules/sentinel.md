## 2026-04-19 - [Enhanced Secret Redaction and Terminal Hardening]
**Vulnerability:** API keys (Anthropic, Stripe, GitHub, JeanBot) were not included in the global `redactSecrets` utility, and the `TerminalService` was exposing raw command output in the `outputPreview` field, potentially leaking sensitive credentials to logs and the UI. Additionally, the terminal command blocklist was missing common RCE patterns like piping to shell interpreters.
**Learning:** Centralized security utilities like `redactSecrets` must be proactively updated as new providers are added, and word boundaries (`\b`) are essential to prevent false positives while ensuring full coverage. Tool services that handle external process output must explicitly apply redaction before persisting display-ready summaries.
**Prevention:** Always apply `redactSecrets` to any output intended for display or audit logs. Use a comprehensive, boundary-aware regex set for all supported AI and platform providers. Maintain a robust, redundant blocklist for terminal command injection patterns.
