## 2026-05-01 - Optimizing Vector Operations and Hashing
**Learning:** High-dimension vector operations (e.g., 1536 dimensions) are significantly slower when using functional patterns like `Array.from`, `reduce`, and `map` compared to standard `for` loops and pre-allocated arrays. Additionally, `Number(val.toFixed(8))` is a hidden performance killer due to string conversion, whereas `Math.round(val * 1e8) / 1e8` provides a massive speedup. Node 22's `crypto.hash` is also ~2x faster than the legacy `createHash` flow.
**Action:** Always prefer `for` loops and pre-allocated arrays for large data sets. Use arithmetic rounding instead of `toFixed` for performance-critical precision math. Explicitly specify 'buffer' as the encoding for `crypto.hash` when binary output is needed for methods like `readUInt32BE`.
